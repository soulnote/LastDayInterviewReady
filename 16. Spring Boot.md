
## **Q1. Differences between Spring and Spring Boot?**

| Aspect | Spring Framework | Spring Boot |
|--------|-----------------|--------------|
| **Purpose** | Dependency Injection, AOP, transaction management | Rapid application development with auto-configuration |
| **Configuration** | Manual XML or Java config | Auto-configuration based on classpath dependencies |
| **Deployment** | Need to configure web.xml, deploy to external server | Embedded server (Tomcat/Jetty/Undertow) - run as JAR |
| **Setup Time** | High (boilerplate code) | Low (starters, defaults) |
| **Use Case** | When you need fine-grained control | Microservices, cloud-native apps |

**  Insight:** Spring Boot is **not a replacement** but an **opinionated wrapper** over Spring Framework.

## **Q9. Explain Transactional Annotation in Spring Boot?**

**Answer:**  
`@Transactional` provides **declarative transaction management** using AOP proxies.

```java
@Service
public class PaymentService {
    
    @Transactional(propagation = Propagation.REQUIRED, 
                   isolation = Isolation.READ_COMMITTED,
                   rollbackFor = Exception.class,
                   timeout = 30)
    public void processPayment(Payment payment) {
        // Multiple DB operations
        // Any RuntimeException causes rollback
    }
}
```

**How it works internally:**
1. Spring creates a **proxy** around the bean
2. Before method execution - begin transaction
3. After method success - commit
4. If exception - rollback
5. After method completion - close transaction

**Important:** Only works on **public methods**. Self-invocation bypasses proxy → no transaction.

## **Q10. What is ResponseEntity in Spring Boot?**

**Answer:**  
`ResponseEntity` represents the **entire HTTP response** - status code, headers, and body.

```java
@GetMapping("/user/{id}")
public ResponseEntity<User> getUser(@PathVariable Long id) {
    User user = userService.findById(id);
    
    if (user == null) {
        return ResponseEntity.notFound().build();
    }
    
    return ResponseEntity.ok()
        .header("Custom-Header", "value")
        .body(user);
}

// Other examples
return ResponseEntity.status(HttpStatus.CREATED).body(newUser);
return ResponseEntity.badRequest().body("Invalid input");
return ResponseEntity.noContent().build();
```

**Difference from `@ResponseBody`:** `@ResponseBody` only returns body with 200 OK. `ResponseEntity` gives full control.

## **Q14 & Q15. Custom exception handling in Spring Boot**

**Answer:** Use `@ControllerAdvice` + `@ExceptionHandler`

```java
// Custom exception
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String message) {
        super(message);
    }
}

// Error response DTO
@Data
@Builder
public class ErrorResponse {
    private LocalDateTime timestamp;
    private int status;
    private String error;
    private String message;
    private String path;
}

// Global exception handler
@ControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(UserNotFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public ResponseEntity<ErrorResponse> handleUserNotFound(UserNotFoundException ex, 
                                                           WebRequest request) {
        ErrorResponse error = ErrorResponse.builder()
            .timestamp(LocalDateTime.now())
            .status(HttpStatus.NOT_FOUND.value())
            .error("User Not Found")
            .message(ex.getMessage())
            .path(((ServletWebRequest)request).getRequest().getRequestURI())
            .build();
        return new ResponseEntity<>(error, HttpStatus.NOT_FOUND);
    }
    
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ResponseEntity<Map<String, String>> handleValidationExceptions(
            MethodArgumentNotValidException ex) {
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getAllErrors().forEach((error) -> {
            String fieldName = ((FieldError) error).getField();
            String errorMessage = error.getDefaultMessage();
            errors.put(fieldName, errorMessage);
        });
        return ResponseEntity.badRequest().body(errors);
    }
    
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public ResponseEntity<ErrorResponse> handleGenericException(Exception ex) {
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(ErrorResponse.builder()
                .timestamp(LocalDateTime.now())
                .status(500)
                .error("Internal Server Error")
                .message(ex.getMessage())
                .build());
    }
}
```

## **Q16. Differences between @MockBean and @SpyBean**

| Aspect | @MockBean | @SpyBean |
|--------|-----------|----------|
| **Purpose** | Complete mock (no real behavior) | Partial mock (real methods unless stubbed) |
| **Default behavior** | Returns null/empty | Calls real method |
| **Method calls** | All methods are mocked | Only stubbed methods are mocked |
| **When to use** | Isolated unit tests, external dependencies | Testing real objects with some methods mocked |
| **Real object** | No real object created | Wraps real Spring bean |

```java
@SpringBootTest
public class UserServiceTest {
    
    @MockBean  // Complete mock
    private UserRepository userRepository;
    
    @SpyBean  // Real service, can stub specific methods
    private EmailService emailService;
    
    @Test
    void testMockBean() {
        // All calls to userRepository return null unless stubbed
        when(userRepository.findById(1L)).thenReturn(Optional.of(user));
    }
    
    @Test
    void testSpyBean() {
        // Email will actually be sent unless stubbed
        doNothing().when(emailService).sendEmail(any()); // Prevent actual send
    }
}
```

## **Q18 & Q20. What is @Async annotation in Spring Boot?**

**Answer:**  
`@Async` enables **asynchronous method execution** using separate threads.

```java
@Configuration
@EnableAsync
public class AsyncConfig {
    
    @Bean
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);
        executor.setMaxPoolSize(10);
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("async-");
        executor.initialize();
        return executor;
    }
}

@Service
public class NotificationService {
    
    @Async
    public CompletableFuture<String> sendEmailAsync(String to, String body) {
        // This runs in separate thread
        // Method returns immediately to caller
        return CompletableFuture.completedFuture("Email sent to " + to);
    }
    
    @Async("taskExecutor") // Custom executor
    public void sendSMS(String number, String message) {
        // No return value - fire and forget
    }
}

// Usage
@RestController
public class Controller {
    @GetMapping("/send")
    public ResponseEntity<String> send() {
        CompletableFuture<String> future = notificationService.sendEmailAsync("test@example.com", "Hello");
        future.thenAccept(result -> log.info(result));
        return ResponseEntity.ok("Request accepted");
    }
}
```

**Limitations:** 
- Only public methods
- Self-invocation doesn't work (no proxy)
- Return type must be void or CompletableFuture

## **Q22. How to integrate SSL certificate in Spring Boot? What is keystore?**

**Answer:**

**Keystore:** A file that stores SSL/TLS certificates and private keys (`.jks`, `.p12`, `.pem`)

**Steps:**

1. **Generate keystore:**
```bash
keytool -genkey -alias tomcat -storetype PKCS12 -keyalg RSA -keysize 2048 \
  -keystore keystore.p12 -validity 3650
```

2. **Configure Spring Boot:**
```properties
# application.properties
server.ssl.enabled=true
server.ssl.key-store=classpath:keystore.p12
server.ssl.key-store-password=yourpassword
server.ssl.key-store-type=PKCS12
server.ssl.key-alias=tomcat
server.port=8443
```

3. **Redirect HTTP to HTTPS:**
```java
@Configuration
public class SSLConfig {
    
    @Bean
    public ServletWebServerFactory servletContainer() {
        TomcatServletWebServerFactory tomcat = new TomcatServletWebServerFactory() {
            @Override
            protected void postProcessContext(Context context) {
                SecurityConstraint securityConstraint = new SecurityConstraint();
                securityConstraint.setUserConstraint("CONFIDENTIAL");
                SecurityCollection collection = new SecurityCollection();
                collection.addPattern("/*");
                securityConstraint.addCollection(collection);
                context.addConstraint(securityConstraint);
            }
        };
        tomcat.addAdditionalTomcatConnectors(redirectConnector());
        return tomcat;
    }
    
    private Connector redirectConnector() {
        Connector connector = new Connector("org.apache.coyote.http11.Http11NioProtocol");
        connector.setScheme("http");
        connector.setPort(8080);
        connector.setSecure(false);
        connector.setRedirectPort(8443);
        return connector;
    }
}
```

## **Q24. Purpose of @ControllerAdvice annotation**

**Answer:**  
`@ControllerAdvice` is a **global interceptor** for controllers - handles exceptions, binds model attributes, and customizes response across all `@Controller`/`@RestController` classes.

```java
@ControllerAdvice
public class GlobalControllerAdvice {
    
    // 1. Global exception handling
    @ExceptionHandler(Exception.class)
    public ResponseEntity<?> handleAllExceptions(Exception ex) { }
    
    // 2. Add global model attributes
    @ModelAttribute
    public void addGlobalAttributes(Model model) {
        model.addAttribute("appVersion", "1.0");
    }
    
    // 3. Bind request data globally
    @InitBinder
    public void customizeBinding(WebDataBinder binder) {
        binder.registerCustomEditor(Date.class, new CustomDateEditor(...));
    }
}
```

**With specific controllers:**
```java
@ControllerAdvice(assignableTypes = {UserController.class})
@ControllerAdvice(basePackages = "com.example.api")
```

## **Q25. Purpose of @ResponseBody annotation**

**Answer:**  
`@ResponseBody` tells Spring to **serialize the return value** directly into HTTP response body (JSON/XML) instead of resolving as view name.

```java
@Controller
public class UserController {
    
    @GetMapping("/user")
    @ResponseBody  // Returns JSON, not "user.jsp"
    public User getUser() {
        return new User("John", "Doe");  // {"firstName":"John", "lastName":"Doe"}
    }
}

// @RestController = @Controller + @ResponseBody
@RestController  // No need @ResponseBody on each method
public class UserRestController {
    @GetMapping("/user")
    public User getUser() {  // Automatically to JSON
        return new User("John", "Doe");
    }
}
```

## **Q27. Property resolution order in Spring Boot**

**Order (highest to lowest priority):**

1. **Command line arguments** `--server.port=8081`
2. **SPRING_APPLICATION_JSON** environment variable
3. **JNDI** attributes (java:comp/env)
4. **Java System properties** `System.getProperties()`
5. **OS environment variables** `SERVER_PORT=8081`
6. **@PropertySource** annotations on `@Configuration` classes
7. **Default properties** from `application.properties` (lowest)

**Profile-specific overrides:**
- `application-{profile}.properties` > `application.properties`
- Profile property has higher priority than default

**YAML vs Properties:** YAML supports nested structures, properties are flatter.

**  Insight:** Use `spring.config.import` for external configs (Config Server, Vault).

## **Q29. Improve performance with caching for frequently fetched data**

**Answer (Multiple caching strategies):**

```java
@Service
public class ProductService {
    
    // Strategy 1: Basic caching
    @Cacheable(value = "products", key = "#id", unless = "#result == null")
    public Product getProduct(Long id) {
        return productRepository.findById(id).orElse(null);
    }
    
    // Strategy 2: Cache with TTL
    @Cacheable(value = "products", key = "#id", 
               cacheManager = "ttlCacheManager")
    public Product getProductWithTTL(Long id) { }
    
    // Strategy 3: Cache eviction on update
    @CacheEvict(value = "products", key = "#product.id")
    public Product updateProduct(Product product) { }
    
    // Strategy 4: Multi-level caching (Caffeine + Redis)
    @Cacheable(value = "products", key = "#id", 
               cacheManager = "multiLevelCacheManager")
    public Product getProductMultiLevel(Long id) { }
}

// Caffeine configuration (in-memory, faster)
@Bean
public CacheManager caffeineCacheManager() {
    CaffeineCacheManager cacheManager = new CaffeineCacheManager("products");
    cacheManager.setCaffeine(Caffeine.newBuilder()
        .expireAfterWrite(5, TimeUnit.MINUTES)
        .maximumSize(1000));
    return cacheManager;
}

// Redis configuration (distributed)
@Bean
public RedisCacheManager redisCacheManager(RedisConnectionFactory factory) {
    RedisCacheConfiguration config = RedisCacheConfiguration.defaultCacheConfig()
        .entryTtl(Duration.ofHours(1))
        .disableCachingNullValues();
    return RedisCacheManager.builder(factory)
        .cacheDefaults(config)
        .build();
}
```

**Caching strategies for different scenarios:**
- **Read-heavy (product catalog):** Cache-aside pattern
- **Write-heavy (inventory updates):** Write-through
- **Near real-time:** TTL-based eviction
- **Distributed cache:** Redis (shared across pods)

## **Q30. What is DDoS and how to implement protection in Spring Boot?**

**Answer:**  
**DDoS** (Distributed Denial of Service) - attackers flood the system with requests to exhaust resources.

**Spring Boot protection strategies:**

### 1. **Rate Limiting (Bucket4j/Resilience4j):**
```java
@Configuration
public class RateLimitingConfig {
    
    @Bean
    public FilterRegistrationBean<RateLimitingFilter> rateLimitingFilter() {
        FilterRegistrationBean<RateLimitingFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new RateLimitingFilter());
        registration.addUrlPatterns("/api/*");
        return registration;
    }
}

public class RateLimitingFilter implements Filter {
    private final Map<String, RateLimiter> limiters = new ConcurrentHashMap<>();
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, 
                         FilterChain chain) throws IOException, ServletException {
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        String clientIp = httpRequest.getRemoteAddr();
        
        RateLimiter rateLimiter = limiters.computeIfAbsent(clientIp, 
            k -> RateLimiter.create(10.0)); // 10 requests per second
        
        if (rateLimiter.tryAcquire()) {
            chain.doFilter(request, response);
        } else {
            HttpServletResponse httpResponse = (HttpServletResponse) response;
            httpResponse.setStatus(429);
            httpResponse.getWriter().write("Too many requests");
        }
    }
}
```

### 2. **Using Resilience4j RateLimiter:**
```xml
<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-spring-boot2</artifactId>
</dependency>
```

```yaml
resilience4j.ratelimiter:
  instances:
    apiRateLimiter:
      limitForPeriod: 100
      limitRefreshPeriod: 1s
      timeoutDuration: 0
```

```java
@RateLimiter(name = "apiRateLimiter")
@GetMapping("/api/data")
public ResponseEntity<?> getData() { }
```

### 3. **Server-level protections:**
```properties
# Tomcat connection limits
server.tomcat.max-connections=10000
server.tomcat.max-threads=200
server.tomcat.accept-count=100
server.tomcat.connection-timeout=20000

# Timeouts
spring.mvc.async.request-timeout=5000
```

### 4. **Custom DDoS mitigation filter:**
```java
@Component
public class DDoSProtectionFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, 
                         FilterChain chain) throws IOException, ServletException {
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        String ip = httpRequest.getRemoteAddr();
        
        // Track requests per IP
        RequestCounter counter = getCounter(ip);
        
        if (counter.incrementAndGet() > 100) { // Threshold
            log.warn("Potential DDoS from IP: {}", ip);
            addToBlocklist(ip);
            ((HttpServletResponse) response).setStatus(429);
            response.getWriter().write("Access denied");
            return;
        }
        
        chain.doFilter(request, response);
    }
}
```

### 5. **Cloud-native protection:**
- Use **API Gateway** (AWS WAF, Cloudflare, Akamai)
- **Rate limiting at load balancer** level
- **Auto-scaling** to absorb traffic

**  Insight:** Rate limiting alone isn't enough - combine with:
- Request size limits
- Slowloris protection (connection timeout)
- Invalid request rejection early (validation)
