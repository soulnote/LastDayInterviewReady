

### **Q1. What is Spring Framework?**
**Answer:**  
Spring ek **lightweight, open-source application framework** hai Java ke liye. Iska main goal **enterprise applications** ko develop karna easy banana hai.  
- **Key features:** Dependency Injection (DI), Aspect-Oriented Programming (AOP), transaction management, MVC web framework, etc.  
- Spring **tight coupling ko reduce** karta hai aur **testability** improve karta hai. Almost every modern Java app Spring Boot (Spring ka extension) use karti hai.

---

### **Q2. What do you understand by DI or Dependency Injection?**
**Answer:**  
DI ek **design pattern** hai jisme object apni dependencies khud create nahi karta, balki **externally inject** karvata hai.  
- **Example:** Agar `Car` class ko `Engine` chahiye, toh `Car` `new Engine()` nahi karegi, balki constructor ya setter se milega.  
- **Benefit:** Loose coupling, easy unit testing.  
- Spring DI ko **IOC container** manage karta hai.

---

### **Q3. What is IOC or Inversion of Control?**
**Answer:**  
IOC ek **principle** hai jisme control flow ko **framework** ke haath mein de diya jata hai, instead of application code.  
- Traditional: Aap khud objects create karte ho.  
- IOC: Spring container decide karta hai kab aur kaunsi object create karni hai.  
- **DI is one way to achieve IOC.**  
- IOC container (e.g., `ApplicationContext`) beans ka lifecycle manage karta hai.

---

### **Q4. What are different modules in spring framework?**
**Answer:**  
Spring bohot modular hai. Main modules:  
1. **Core Container:** `Core`, `Beans`, `Context`, `SpEL`  
2. **AOP:** Aspect-oriented programming  
3. **Data Access:** JDBC, ORM (Hibernate), Transactions  
4. **Web:** Spring MVC, WebSocket, WebFlux (reactive)  
5. **Integration:** JMS, Email, Scheduling  
6. **Test:** Mock objects, TestContext framework  

 Aap usually `spring-boot-starter-web`, `spring-boot-starter-data-jpa` use karte ho, jo internally in modules ko pull karte hain.

---

### **Q5. Explain the bean scopes supported by Spring?**
**Answer:**  
1. **singleton** (default) ŌĆō ek bean definition per container ek instance.  
2. **prototype** ŌĆō har request par naya instance.  
3. **request** ŌĆō ek HTTP request ke liye ek bean (web-aware context).  
4. **session** ŌĆō ek HTTP session ke liye ek bean.  
5. **application** ŌĆō `ServletContext` ke liye ek bean.  
6. **websocket** ŌĆō WebSocket ke liye.  

 `singleton` thread-safe nahi hai by default; aapko state management khud karni hoti hai.

---

### **Q6. Explain Spring Bean Life Cycle?**
**Answer:**  
1. **Instantiate** ŌĆō constructor call.  
2. **Populate properties** ŌĆō setter injection.  
3. **BeanNameAware** ŌĆō bean name set.  
4. **BeanFactoryAware** ŌĆō factory set.  
5. **Pre-initialization** ŌĆō `BeanPostProcessor` ka `postProcessBeforeInitialization`.  
6. **InitializingBean** ŌĆō `afterPropertiesSet()`.  
7. **Custom init method** ŌĆō `@PostConstruct` ya `init-method`.  
8. **Post-initialization** ŌĆō `postProcessAfterInitialization`.  
9. **Bean ready to use**.  
10. **Destroy** ŌĆō `@PreDestroy` ya `DisposableBean` ka `destroy()`.

 `@PostConstruct` aur `@PreDestroy` sabse commonly use hote hain.

---

### **Q7. Explain different types of Autowiring?**
**Answer:**  
Spring 5+ mein mostly **annotation-based** autowiring use hoti hai. XML types:  
1. **no** ŌĆō no autowiring.  
2. **byName** ŌĆō property name se bean match.  
3. **byType** ŌĆō type match (agar ek se zyada same type toh error).  
4. **constructor** ŌĆō constructor arguments autowire.  
5. **autodetect** ŌĆō (deprecated).  

**Modern way:** `@Autowired`, `@Inject`, `@Resource`.

 `@Autowired(required=false)` se optional injection.

### **Q8. What is the difference between Tight Coupling and Loose Coupling?**
| Tight Coupling | Loose Coupling |
|----------------|----------------|
| Object directly creates dependency (`new Engine()`) | Dependency injected externally |
| Hard to test | Easy to mock |
| Change in one class affects many | Changes are isolated |
| Example: `Car` depends on `PetrolEngine` class directly | `Car` depends on `Engine` interface |

 Loose coupling **Dependency Inversion Principle** follow karta hai (SOLID ka ŌĆśDŌĆÖ).

### **Q10. Which is the default scope of bean in Spring? and, are they synchronized?**

-  Stateless beans (like `@Service`, `@Repository`) safe hain. Stateful beans mein `@Scope("prototype")` use karo ya thread-local storage.

### **Q15. What is the purpose of the @Qualifier annotation in Spring?**


### **Q20. Explain Spring Transaction Propagation Levels?**
**Answer:**  
Propagation defines **how a method behaves when called within an existing transaction**.  
1. **REQUIRED** (default) ŌĆō Join existing, else create new.  
2. **REQUIRES_NEW** ŌĆō Always suspend current, create new.  
3. **NESTED** ŌĆō Savepoint inside existing transaction (JDBC 3.0+).  
4. **MANDATORY** ŌĆō Existing must be there, else exception.  
5. **SUPPORTS** ŌĆō If exists, join; else non-transactional.  
6. **NOT_SUPPORTED** ŌĆō Suspend current, run non-transactional.  
7. **NEVER** ŌĆō Exception if transaction exists.

 `REQUIRES_NEW` carefully use karo ŌĆō it commits independently, which can cause unexpected rollbacks.

### **Q28. Which Is the Best Way of Injecting Beans?**
**Answer:**

- Immutability (final fields)  
- No reflection overhead (field injection uses reflection)

### **Q29. Which DI would you suggest Constructor-based or setter-based DI?**
**Answer:**

- **Setter-based** ŌĆō optional dependencies ke liye.

### **Q30. How does Transactional annotation works internally?**
**Answer:**  
`@Transactional` internally **AOP proxy** create karta hai:  
1. Spring creates a proxy of the bean.  
2. When method called, proxy starts a transaction (using `PlatformTransactionManager`).  
3. Method executes.  
4. If no exception, commit; if `RuntimeException` or specified exception, rollback.  
5. Transaction is closed.  

 **Important** ŌĆō Only works on `public` methods. Internal method calls bypass proxy ŌåÆ transaction nahi lagegi. Use `self-injection` or `TransactionTemplate`.

---

### **Q31. Explain the usage of the @Transactional annotation in Spring?**
**Answer:**  
- Used on **service layer methods** to manage database transactions declaratively.  
- Example:  
```java
@Transactional
public void transferMoney(Account from, Account to, BigDecimal amount) {
    from.debit(amount);
    to.credit(amount);
}
```
- Customize: `rollbackFor`, `noRollbackFor`, `propagation`, `isolation`, `timeout`.  
-  Always use `@Transactional` on **public methods** only. `@Transactional` on class level applies to all methods. Avoid `@Transactional` on `private` or `final` methods (proxy wonŌĆÖt work).

## **Q1. Develop a caching mechanism for your application to boost performance and decrease database load?**

**Answer:**  
Spring provides **declarative caching** using `@Cacheable`, `@CacheEvict`, `@CachePut`.

**Implementation:**
```java
@Service
public class UserService {
    
    @Cacheable(value = "users", key = "#id", unless = "#result == null")
    public User getUserById(Long id) {
        // Database call - will be cached
        return userRepository.findById(id).orElse(null);
    }
    
    @CacheEvict(value = "users", key = "#user.id")
    public User updateUser(User user) {
        return userRepository.save(user);
    }
    
    @CachePut(value = "users", key = "#result.id")
    public User createUser(User user) {
        return userRepository.save(user);
    }
}
```

**Configuration:**
```java
@Configuration
@EnableCaching
public class CacheConfig {
    
    @Bean
    public CacheManager cacheManager() {
        return new ConcurrentMapCacheManager("users", "products");
    }
}
```

**SDE-2 insights:**
- Use **Redis** for distributed caching (multiple pods)
- Set **TTL** (Time-To-Live) using `@Cacheable(value="users", key="#id", cacheManager="redisCacheManager")`
- **Cache Eviction strategy:** `@CacheEvict(allEntries=true)` for bulk updates
- **Cache Aside pattern** implement karo - application manages cache directly

## **Q2. Suppose you're getting an information on every request in controller level, how can you utilize that particular info throughout the project?**

**Answer:** Use **Request Scope** + **Interceptor** + **ThreadLocal**.

**Solution:**
```java
@Component
@Scope(value = "request", proxyMode = ScopedProxyMode.TARGET_CLASS)
public class RequestContext {
    private String correlationId;
    private String userId;
    private Map<String, Object> attributes = new HashMap<>();
    // getters/setters
}

// Interceptor to populate
@Component
public class CustomInterceptor implements HandlerInterceptor {
    @Autowired
    private RequestContext requestContext;
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, Object handler) {
        requestContext.setCorrelationId(request.getHeader("X-Correlation-Id"));
        requestContext.setUserId(request.getHeader("X-User-Id"));
        return true;
    }
}

// Use anywhere
@Service
public class BusinessService {
    @Autowired
    private RequestContext requestContext;
    
    public void process() {
        String userId = requestContext.getUserId(); // Available throughout
    }
}
```

**SDE-2 alternative:** Use **MDC (Mapped Diagnostic Context)** for logging:
```java
MDC.put("correlationId", UUID.randomUUID().toString());
// Access anywhere: MDC.get("correlationId")
```

## **Q3 & Q5. Scheduled cron job on multiple pods - how to avoid duplicate work?**

**Answer (without external tools):** Use **Database Locking** or **ShedLock**.

**Solution 1 - ShedLock (Recommended):**
```java
// Add dependency: shedlock-spring
@Configuration
@EnableScheduling
@EnableSchedulerLock(defaultLockAtMostFor = "PT30S")
public class SchedulerConfig {
    
    @Bean
    public LockProvider lockProvider(DataSource dataSource) {
        return new JdbcTemplateLockProvider(dataSource);
    }
}

@Component
public class MyScheduler {
    
    @Scheduled(cron = "0 0 2 * * ?") // 2 AM daily
    @SchedulerLock(name = "nightlyJob", 
                   lockAtMostFor = "PT1H", 
                   lockAtLeastFor = "PT5M")
    public void runNightlyJob() {
        // Only one pod will execute this
    }
}
```

**Solution 2 - With Aspect + Apache Curator (ZooKeeper):**
```java
@Aspect
@Component
public class DistributedLockAspect {
    @Autowired
    private CuratorFramework curatorClient;
    
    @Around("@annotation(DistributedLock)")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        String lockPath = "/locks/" + joinPoint.getSignature().toShortString();
        InterProcessMutex lock = new InterProcessMutex(curatorClient, lockPath);
        
        try {
            if (lock.acquire(10, TimeUnit.SECONDS)) {
                return joinPoint.proceed();
            } else {
                log.info("Another instance is executing this job");
                return null;
            }
        } finally {
            lock.release();
        }
    }
}
```

## **Q4. Run scheduled tasks only on specified servers**

**Answer:** Use **Conditional scheduling** with environment properties.

```java
@Configuration
@EnableScheduling
public class ScheduledConfig {
    
    @Value("${scheduling.enabled:false}")
    private boolean schedulingEnabled;
    
    @Value("${scheduling.server.id}")
    private String currentServerId;
    
    @Value("${scheduling.master.server.id}")
    private String masterServerId;
    
    @Bean
    public Condition schedulingCondition() {
        return schedulingEnabled && currentServerId.equals(masterServerId);
    }
}

@Component
public class ConditionalScheduler {
    
    @Scheduled(cron = "0 0 * * * *")
    @ConditionalOnProperty(name = "scheduling.enabled", havingValue = "true")
    @ConditionalOnExpression("'${scheduling.server.id}'.equals('${scheduling.master.server.id}')")
    public void scheduledTask() {
        // Runs only on master server
    }
}
```

**Alternative:** Use **Spring Profiles**:
```bash
# On master server:
java -jar app.jar --spring.profiles.active=master

# On other servers:
java -jar app.jar --spring.profiles.active=slave
```

```java
@Component
@Profile("master")
public class MasterOnlyScheduler {
    @Scheduled(cron = "0 0 * * * *")
    public void task() { }
}
```

## **Q6 & Q7. Validate incoming JSON request body schema**

**Answer (without external validator):** Use **Spring Validation** + **Custom Validators**.

```java
// DTO with validation annotations
public class UserRequest {
    
    @NotNull(message = "firstName cannot be null")
    @Pattern(regexp = "^[A-Za-z]*$", message = "firstName must contain only letters")
    private String firstName;
    
    @NotNull
    @Email
    private String email;
    
    @Min(18)
    @Max(120)
    private Integer age;
    
    // Custom validation for dynamic field names
    @AssertTrue(message = "Either firstName or first_Name must be present")
    public boolean isValidName() {
        return firstName != null; // Handle first_Name in custom deserializer
    }
}

// Custom deserializer for flexible field names
public class FlexibleNameDeserializer extends StdDeserializer<String> {
    @Override
    public String deserialize(JsonParser p, DeserializationContext ctxt) 
            throws IOException {
        ObjectCodec codec = p.getCodec();
        JsonNode node = codec.readTree(p);
        
        if (node.has("firstName")) {
            return node.get("firstName").asText();
        } else if (node.has("first_Name")) {
            return node.get("first_Name").asText();
        }
        return null;
    }
}

// Controller
@PostMapping("/user")
public ResponseEntity<?> createUser(@Valid @RequestBody UserRequest request) {
    // Automatic validation happens here
}
```

**Type mismatch handling:**
```java
@RestControllerAdvice
public class GlobalExceptionHandler {
```

## **Q9. API should produce JSON for React and XML for another microservice**

**Answer:** Use **Content Negotiation** (Accept header).

```java
@RestController
public class TransactionController {
    
    @GetMapping(value = "/transactions", 
                produces = {MediaType.APPLICATION_JSON_VALUE, 
                           MediaType.APPLICATION_XML_VALUE})
    public List<Transaction> getTransactions() {
        return transactionService.findAll();
    }
}

// Configuration
@Configuration
public class ContentNegotiationConfig implements WebMvcConfigurer {
    
    @Override
    public void configureContentNegotiation(ContentNegotiationConfigurer configurer) {
        configurer
            .favorParameter(true)
            .parameterName("format")
            .ignoreAcceptHeader(false)
            .defaultContentType(MediaType.APPLICATION_JSON)
            .mediaType("json", MediaType.APPLICATION_JSON)
            .mediaType("xml", MediaType.APPLICATION_XML);
    }
}
```

**Usage:**
- React: `GET /transactions` ŌåÆ JSON
- Microservice: `GET /transactions?format=xml` ŌåÆ XML
- Or use Accept header: `Accept: application/xml`

**For XML support, add Jackson XML dependency:**
```xml
<dependency>

</dependency>
```

## **Q10. Service crashes with OOM error - How to debug?**

**Answer:** Step-by-step debugging approach:

1. **Enable Heap Dump on OOM:**
   ```bash
   java -XX:+HeapDumpOnOutOfMemoryError 
        -XX:HeapDumpPath=/logs/heapdump.hprof 
        -jar app.jar
   ```

2. **Analyze heap dump using MAT (Memory Analyzer Tool):**
   - Find biggest objects: `Retained Heap` size check karo
   - Check for **memory leaks**: `Leak Suspects Report`
   - Find **GC roots** for retained objects

3. **Common causes & solutions:**
   - **Unbounded cache:** Implement `@CacheEvict` or TTL
   - **Large collections:** Use pagination, streaming
   - **Thread pool exhaustion:** Configure proper pool sizes
   - **File/Stream not closed:** Use try-with-resources

4. **Monitoring during runtime:**
   ```java

   public class MemoryMonitor {
       @Scheduled(fixedDelay = 60000)
       public void logMemory() {
           MemoryMXBean memoryBean = ManagementFactory.getMemoryMXBean();
           MemoryUsage heapUsage = memoryBean.getHeapMemoryUsage();
           log.info("Heap: {} MB used / {} MB max",
               heapUsage.getUsed() / 1048576,
               heapUsage.getMax() / 1048576);
       }
   }
   ```

5. **Tools:** JProfiler, VisualVM, YourKit, Eclipse MAT

## **Q13. Scheduled method that triggers POST endpoint monthly with parameters**

```java
    private RestTemplate restTemplate;

            .id(UUID.randomUUID().toString())
            .l4(5)
            .l5(0)

            request,
            String.class

        }
    }
}

    private String id;

    private Integer l5;
    private Integer l6;
}
```

```java
    public void run(ApplicationArguments args) {

        System.out.println("Total beans: " + beanNames.length);

            .filter(name -> !name.startsWith("org.springframework"))
```
